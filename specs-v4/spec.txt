// symbols

'@' : macro. replaces something.
'#' : directive. tells compiler something
'*' : multiplication. can be used for arithmetic, and also for arrays, as arrays means multiple same data.
'&' : mutable
'()' : expression. inside this parenthesis are considered as one expression.
'-' : minus. used for arithmetic.
'+' : plus. used for arithmetic.
'=' : assignment. means moving data.
'{}' : aggregate. bundles data into one aggregate. it can be arrays, structs, bitfields.
'[]' : memory operation. load and store to memory.
'=>' : call. branch to function.
'->' : jump. branch to location.
'>>' : jump to matching << in scope. may be depricated in future.
',' : next register. also implies dependency break, as these comma separated expressions cannot refer to each other.
'.' : offset. X.Y.Z means hierarchy, and is never interpreted as multiple expression.
':' : right to left. inverts assignment directions. when used solo, binds static location to a name.
'::' : named variable declaration. right to left. binds dynamic location to name(s).
''' : character literal.
'"' : string view. it is a readonly view to the string embedded in the code.
'?' : conditional true.
'!' : conditional false.
'`' : formatting string.
'>' '<' '>=' '<=' : comparison. sets flag.
':' and compound : (e.g. ':=' ':+') operates to the left hand side.
'_' : private. don't use this unless you understand internals. these symbols are not exported.

// punctuations, not fixed
'$' : explicit register. 
'<>' : aggregate view.
'~' : owned address annotation
'~>' : ownership transfer
'%' : move to matching %
';' : end of line
'|' : parallel
'^' : outer scope
'\' : escaping | cast


// reserved keywords

ret : returns from function.
struct : declares a struct.
enum : declares an enum.
union : declares an union.
undefined : undefined state.
unreachable: should be unreachable. panics if this code is reached. if "#optimize unreachable" is on, reaching this code will be undefined behaviour.
is : check equality

# constant/variable and mutable/immutable

constants are decided at compile time.
variables are decided at runtime.

all constants are immutable.
variables can be mutable or immutable.
